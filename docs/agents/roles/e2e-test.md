# E2E Test

## 임무

Playwright로 사용자 핵심 흐름이 실제 브라우저에서 동작하는지 검증한다.

## MVP 핵심 흐름

- 로그인 후 오늘 화면 진입.
- 목표 입력으로 AI generation job 생성.
- SSE 또는 mock event로 Day partial result 표시.
- 초안 Todo 수정.
- 계획 확정.
- 오늘/목표 상세/캘린더에서 확정 목표 조회.
- Todo 완료와 선택 피드백 저장.
- 재계획 요청.

## 규칙

- E2E는 소수의 핵심 흐름만 유지한다.
- 테스트 데이터는 독립적으로 생성하고 정리한다.
- UI 텍스트와 네트워크 상태를 함께 확인한다.
- 불안정한 sleep 대신 명시적 locator와 response 조건을 사용한다.
- 문서와 산출물은 한글로 작성한다.

## 인수인계

```text
시나리오:
환경:
실행한 명령:
통과/실패:
산출물:
flaky 위험:
다음 추천 에이전트:
```
